第一百一十三章 聊聊 jQuery

我们为什么要学习 jQuery，它有什么用。

我们前面学习了一个叫做函数的东西，它可以让我们把重复的操作作为一个整体，在需要的时候叫它一下，便可以执行这一系列的操作。这种方法让许多的情境变得简单。

如果进一步思考，我们对于页面执行的操作，总有一些很基础的，又非常常用的。如果我们把这些东西写成函数，放在一个文件之中，在我们的页面中引入，那么后面的开发，是不是就省去了一部分重复的操作。

这就像我们在工作中，都喜欢准备一套基础的工具，因为经常会用到，有这么一套工具，总会让工作变得简单和顺手一些。

jQuery 是一个 JS 库，库呢，差不多就可以理解为我们上面所说的工具箱。换句话说，jQuery 就是让我们对页面的基本操作变得更加简单。

讲到这里，大家一定觉得这是一个非常有意义的东西。在以前来说算是，而现在就不太好给出一个确切的定义了。网上很多人讲，jQuery 早就已经淘汰了，这个观点，我不太承认。确实我们现在有很多新的解决问题的方法，总体的思路都已经和以前大不相同，在这个方向上，可能 jQuery 的用武之地已经非常少了。可是我们有了电动机械之后，也不能说手动机械就彻底淘汰了。日常敲颗钉子，我还是使用已经用了千百年的锤子，就觉得很顺手。而且在这样简单的情境下，这种落后的工具已经可以很好的完成任务了，我也没必要为了表明自己的先进性，去搞一套复杂的设备，来敲这一颗钉子。

还有一个方面，JS 也在快速的成长，在现在的 JS 标准下，已经有很多方法可以很好的去替代 jQuery 的操作了[1]。我甚至纠结了很长时间，现在我是不是很有必要给大家讲 jQuery。

直到我想到了一个比喻：如果把 JS 比作刀子，那 jQuery 就是转笔刀。刀子是基础工具，它很简单，甚至于简陋，但它可以做的事情却非常的多，只要你敢想，只要你有足够的手艺，它就可以辅助你去完成各种有趣的加工。如果你想做一个出色的手艺人，那就必须把这把刀子玩转。

转笔刀就差远了，它能做的事情只有这一件，就是削铅笔。削铅笔也削不出来花，削出来的永远都是那一个样子。真的是逊到让人不想看。可是这样转笔刀就没有它存在的意义吗？我上小学的时候手很笨，铅笔削出来像是狗啃的，一点也不好用。而父母坚持让我使用刀子削铅笔，不肯给我转笔刀。平时也就算了，直到那次考试的时候，铅笔芯折了……

我们现在就是初学者，而我的教程呢，就是带着大家对这些技术观光一番，先了解前端是什么，有什么，能做什么。我可以给大家讲各种非常先进帅气的方法，但是那样要学习的内容太多了。我们会在一个地方停留太久，大量的记忆就意味着枯燥，而抬头看看，发现自己并没有向前走出多远，这很容易让人想放弃。

我们管它这个方法究竟 low 不 low 呢？如果我可以用一句话的语法对付成片的问题，那我先用起来，走到远方去看一看，然后再选择自己应该怎样去走这段路。毕竟付出并不高，却可以让我先了解前面的方向。所以为什么不学一学呢？
