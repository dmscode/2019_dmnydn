第二十二章 一些常用的标签

对于 HTML5 中新增的，常用的语义化标签的简要介绍。

有没有这样一种感觉？就是每一章都能够听得懂，但是回想一下，却又不知道如何去用。就仿佛虽然双脚实实在在的站在地上，但是放眼看向四周，全是白茫茫的雾气，竟然一时分不清了上下。

不要着急，也不要担心，这是非常正常的一件事情。因为 HTML 并不是一个孤立的存在，现在对这些语言已经有了非常明确的分工，他们之间要相互配合才能完成我们所期望的任务。而现在，我们单单讲一个方面，随着不断的深入，自然会越来越明显的感受到周围未知区域的存在。不过这不是什么问题，因为随着我们内容的展开，一切自然而然会变得明朗。

目前呢，你就当做我一句话还没有说完，不要急着下什么判断，安安静静，听我把故事从头说到尾。中间可不许瞌睡哦~

这一章我们再来简单的介绍一些常用的标签。大家认真看一看，先有个了解，后面随着我们的使用，再去逐渐熟悉。

<header> 页头

这个标签内，放网站的页头内容，比如常见的，logo、标题、导航等。

<footer> 页脚

网站最底下用来显示一些链接，版权信息等的部分。

<nav> 导航

这个标签里面用来放导航内容。

<aside> 侧边栏

这里面放侧边栏的内容，但说是侧边栏，其实只是为了让大家更好理解。它并不一定非要在侧边。

<main> 主体

页面中的主体内容

<article> 文章

文章内容，可以粗略的理解为，一篇文章。

<section> 章节

一个小节，或者说一个小的章节。但是要注意这里我们所说的文章，可能并不是我们传统意义上的一篇文章，也许更应该说它是一个整体的内容，而章节则是对这一个整体内容的进一步细分。

很容易发现，我们今天讲的这些标签就是用来描述一个页面基本结构的。这些标签全部都是 HTML5 新增加的，他们并不会产生什么特别的效果，非常单纯的就是在说明被标记的内容是什么。所以我们也可以看出 HTML5 非常重视用标签把每一项内容是什么说清楚。

这件事情做好之后的结果是什么呢？我们的代码变得更容易被阅读，虽然只是代码，但是看一下标签，我就能分出页头、页尾、主体内容和侧边栏，看到文章的主体内容还可以继续分割到小节，可以说这篇文章的提纲，马上就了然于胸了。这比用多层表格嵌套强行布局出来的内容，读起来要舒适太多。

好了，这一章就到这里，后面我们要为学习 CSS 做准备工作了，是不是有点小激动呢~
